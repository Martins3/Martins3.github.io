# 用写作来重新思考问题

## 为什么我开始写
首先，基于一些朴素的恨意:

1. 我高中的时候，每次语文作文都打了一个很低的分数。
我问我的语文老师: 为什么我们非要浪费时间写一些毫无意义的东西，这些作文既没有表达什么思想，也没有高效的记录内容。
语文老师反问我: 那你有你觉得比较好的形式的作文吗，可以拿出来给我看一下。
当时我不知道如何作答，那么现在我可以说，这个仓库就是我对于"作文"的理解，不要废话，迎合，形式，神秘主义，800 字，只可意会，只有记录，分析和表达。

2. CSDN 之类的毒瘤，鼓励抄袭盗版，把中文社区更加是乌烟瘴气。改善这种情况，我认为每一个都从自己做起，尽量输出一些自己的原创内容。

此外，写 blog 可以讲经常查询的东西记录下来:
1. 很多小细节记录了，之后免得重新再次 Google 或者问 ChatGPT ，类似这个
	- https://github.com/jbranchaud/til
	- https://github.com/chenzhiwei/linux

最关键的作用是，在写的过程中，我在重新思考问题。

## 写作的真正力量

1. 在写之前，我以为我已经完全懂了，但是写的过程中，往往发现很多事情都不懂。
将想法转化为语言迫使我澄清我的理解。这种清晰表达的过程有助于识别理解可能模糊或不完整的领域。

2. 在某些情况下，撰写有关某个主题的文章可能需要对该主题进行全面的探索。这通常会导致发现新的见解和联系，而这些见解和联系可能仅通过阅读或聆听而无法显现出来。

我在写 [Kernel Bypass](https://martins3.github.io/kernel/why-by-pass.html) 的总结的时候，以为自己已经很清楚了，但是后面发现了
[Linux drivers in user space — a survey](https://lwn.net/Articles/703785/) 中的内容，意识到自己其实忽视了很大的一片内容。
